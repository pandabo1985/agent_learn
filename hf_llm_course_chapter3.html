01

本章简介 从「预测」到「训练」

前两章我们用 Tokenizer + 预训练模型完成了推理。但仅用两三句话训练模型不会有好效果——本章教你加载真实数据集、微调出可用的模型。

02

加载数据集 load_dataset / DatasetDict

🤗 Datasets 提供一条便捷命令，可在 Hub 上下载并缓存数据集：

from datasets import load_dataset

raw_datasets = load_dataset("glue", "mrpc")
raw_datasets

返回的是一个 DatasetDict，内部含训练 / 验证 / 测试三个划分，每个划分 4 列：

DatasetDict({
    train:      Dataset({ features: ['sentence1', 'sentence2', 'label', 'idx'], num_rows: 3668 })
    validation: Dataset({ features: ['sentence1', 'sentence2', 'label', 'idx'], num_rows: 408  })
    test:       Dataset({ features: ['sentence1', 'sentence2', 'label', 'idx'], num_rows: 1725 })
})
💾
缓存位置 默认下载到 ~/.cache/huggingface/datasets；可用 HF_HOME 环境变量自定义（同第二章的模型缓存机制）。

查看单条样本与 features

raw_train_dataset = raw_datasets["train"]
raw_train_dataset[0]
# {'idx': 0, 'label': 1,
#  'sentence1': 'Amrozi accused his brother ...',
#  'sentence2': 'Referring to him as only " the witness " ...'}

标签已经是整数，无需再预处理。若想知道整数对应的真实含义，查看 .features：

raw_train_dataset.features["label"]
# ClassLabel(num_classes=2, names=['not_equivalent', 'equivalent'])
# 0 → not_equivalent（非同义），1 → equivalent（同义）

🔢 ClassLabel

建立整数 ↔ 类别的映射。label 已是整数，可直接喂给模型做训练。

🗂️ Apache Arrow

数据集以 Arrow 格式存于磁盘，按需加载到内存，不会一次性撑爆内存。

03

预处理与分词 token_type_ids / map()

要把文本转成模型能理解的数字（第二章已学）。本任务特殊之处：每个样本是一对句子，需让模型区分哪部分是第一句、哪部分是第二句。Tokenizer 支持直接输入句子对：

from transformers import AutoTokenizer
checkpoint = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

inputs = tokenizer("This is the first sentence.", "This is the second one.")
# {'input_ids':      [101, 2023, ..., 1012, 102, 2023, ..., 1012, 102],
#  'token_type_ids': [0,   0,    ..., 0,    0,   1,    ..., 1,    1],
#  'attention_mask': [1,   1,    ..., 1,    1,   1,    ..., 1,    1]}

token_type_ids：第二章没讲的新字段

它的作用是告诉模型「输入哪部分是第一句、哪部分是第二句」。把 input_ids 还原成 token 后，结构一目了然：

BERT 句子对格式：[CLS] 句子1 [SEP] 句子2 [SEP]
[CLS]thisisthefirstsentence.[SEP] thisisthesecondone.[SEP]
00000000 1111111 ← token_type_ids
💡
不是所有模型都有 token_type_ids 它来自 BERT 预训练的「下一句预测（NSP）」任务。DistilBERT 就不会返回。只要 tokenizer 与模型来自同一 checkpoint，tokenizer 自己知道该提供什么，无需操心。

用 map() 处理整个数据集

直接把整列句子喂给 tokenizer 虽可行，但会一次性把整个数据集读入内存。更好的做法是定义函数，配合 Dataset.map()：

def tokenize_function(example):
    return tokenizer(example["sentence1"], example["sentence2"], truncation=True)

tokenized_datasets = raw_datasets.map(tokenize_function, batched=True)

batched=True

一次处理多个元素，配合 Rust 编写的快速 tokenizer，显著提速。

追加字段

返回字典的 key（input_ids 等）会作为新列加入数据集，原列保留。

暂不 padding

把全部样本填到最大长度很浪费——填充留到组 batch 时做。

⚠️
关键决定：不在 tokenize_function 里 padding 在构建 batch 时才填充，只需补齐到「该 batch 的最大长度」，而非整个数据集的最大长度。这就是下一节的动态填充。
04

动态填充 collate 函数 / DataCollatorWithPadding

负责把一个 batch 里的数据整理成张量的函数叫 collate 函数。默认实现会把元素拼接成张量，但我们的句子长度不一，无法直接堆叠。于是刻意延迟填充，只在每个 batch 上补齐到该批最长：

策略做法代价
一次性填充到最大长度 整个数据集都填到全局最长（或模型上限 512） 大量无用 token，浪费算力
动态填充（推荐） 每个 batch 只补齐到该批最长 大幅节省时间；但张量形状逐批变化

🤗 Transformers 用 DataCollatorWithPadding 直接提供这个函数，只需传入 tokenizer（它要知道用哪个 pad token、左填还是右填）：

from transformers import DataCollatorWithPadding
data_collator = DataCollatorWithPadding(tokenizer=tokenizer)

验证动态填充效果

抽 8 个样本，去掉字符串列（无法转张量），看各样本原始长度，再过一遍 collator：

samples = tokenized_datasets["train"][:8]
samples = {k: v for k, v in samples.items() if k not in ["idx", "sentence1", "sentence2"]}
[len(x) for x in samples["input_ids"]]
# [50, 59, 47, 67, 59, 50, 62, 32]  ← 长度 32 到 67 不等

batch = data_collator(samples)
{k: v.shape for k, v in batch.items()}
# {'attention_mask': torch.Size([8, 67]),   ← 全部补齐到 67（本批最大）
#  'input_ids':      torch.Size([8, 67]),
#  'token_type_ids': torch.Size([8, 67]),
#  'labels':         torch.Size([8])}        ← 注意：data_collator 还把 label 改名为 labels
⚠️
TPU 特例 动态填充让张量形状逐批变化，但 TPU 偏好固定形状。在 TPU 上建议改用 padding="max_length" + max_length，即便要填一些无用 token。

至此，原始文本已转化为模型可处理的数据，可以开始微调了。

05

Trainer API 微调 高层封装 / compute_metrics

🤗 Transformers 的 Trainer 帮你在数据集上微调任意预训练模型。完成预处理后，定义 Trainer 只需三步：

① TrainingArguments —— 所有超参数

from transformers import TrainingArguments
training_args = TrainingArguments("test-trainer")  # 参数=保存模型与 checkpoint 的目录

② 加载模型（替换分类头）

from transformers import AutoModelForSequenceClassification
model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
🐞
会收到警告——这是正常的！ BERT 没有针对「句子对分类」预训练过，所以预训练的旧 head 被丢弃，换上一个适合序列分类的新头（随机初始化）。警告说明：旧 head 权重未使用、新 head 权重随机初始化。这正是「微调」的起点。

③ 组装 Trainer 并训练

from transformers import Trainer

trainer = Trainer(
    model, training_args,
    train_dataset=tokenized_datasets["train"],
    eval_dataset=tokenized_datasets["validation"],
    data_collator=data_collator,
    tokenizer=tokenizer,
)
trainer.train()
⚠️
第一次训练看不到指标？ 默认每 500 步只报告训练 loss。要看到性能，需：① 设 evaluation_strategy="epoch" 让它评估；② 提供 compute_metrics 函数（否则只报 loss，不直观）。

④ 构造 compute_metrics()

该函数接收 EvalPrediction（含 predictions 与 label_ids），返回「指标名 → 值」的字典。先看模型预测结果：

predictions = trainer.predict(tokenized_datasets["validation"])
# 返回 (predictions, label_ids, metrics)；predictions 形状 408×2（408 样本 × 2 标签的 logits）

import numpy as np
preds = np.argmax(predictions.predictions, axis=-1)  # 取每行最大值的索引 → 预测类别
import evaluate
def compute_metrics(eval_preds):
    metric = evaluate.load("glue", "mrpc")
    logits, labels = eval_preds
    predictions = np.argmax(logits, axis=-1)
    return metric.compute(predictions=predictions, references=labels)

# 返回 {'accuracy': 0.858, 'f1': 0.900} —— 与 BERT 论文基础模型 F1=88.9 相当或更好

把 compute_metrics 传入新的 Trainer，并设 evaluation_strategy="epoch"，每个 epoch 结束就会报告验证 loss 与指标。

🔑
小贴士 重新训练前要新建模型，否则会在已训练过的模型上继续训。Trainer 支持多 GPU/TPU 与混合精度（fp16=True），更多选项在第十章展开。
06

完整训练循环 不用 Trainer，手写 PyTorch

现在脱离 Trainer，用原生 PyTorch 实现同样的训练效果。这要求我们手动完成 Trainer 自动处理的三件事。

① 数据后处理 + DataLoader

tokenized_datasets = tokenized_datasets.remove_columns(["sentence1", "sentence2", "idx"])
tokenized_datasets = tokenized_datasets.rename_column("label", "labels")  # 模型默认输入叫 labels
tokenized_datasets.set_format("torch")  # 返回 PyTorch 张量而非列表
# 现在只剩：['attention_mask', 'input_ids', 'labels', 'token_type_ids']

from torch.utils.data import DataLoader
train_dataloader = DataLoader(tokenized_datasets["train"], shuffle=True, batch_size=8, collate_fn=data_collator)
eval_dataloader   = DataLoader(tokenized_datasets["validation"], batch_size=8, collate_fn=data_collator)

② 模型 + 优化器 + 学习率调度器

from transformers import AutoModelForSequenceClassification, get_scheduler
from torch.optim import AdamW

model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
optimizer = AdamW(model.parameters(), lr=5e-5)  # Adam + 解耦权重衰减

num_epochs = 3
num_training_steps = num_epochs * len(train_dataloader)
lr_scheduler = get_scheduler("linear", optimizer=optimizer,
                             num_warmup_steps=0, num_training_steps=num_training_steps)
# 默认调度：学习率从最大值（5e-5）线性衰减到 0
📐
当输入含 labels 时，模型直接返回 loss outputs = model(**batch) 会得到 outputs.loss 和 outputs.logits（形状 8×2）。所以训练循环里不必自己算损失。

③ 训练循环：每个 batch 的四步

STEP 1
outputs = model(**batch)
前向传播，得 loss + logits
STEP 2
loss.backward()
反向传播，计算梯度
STEP 3
optimizer.step()
用梯度更新权重
STEP 4
optimizer.zero_grad()
清空梯度，防累积
import torch
from tqdm.auto import tqdm

device = torch.device("cuda") if torch.cuda.is_available() else torch.device("cpu")
model.to(device)
progress_bar = tqdm(range(num_training_steps))

model.train()
for epoch in range(num_epochs):
    for batch in train_dataloader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)
        loss = outputs.loss
        loss.backward()

        optimizer.step()
        lr_scheduler.step()       # 别忘了更新学习率
        optimizer.zero_grad()
        progress_bar.update(1)

④ 评估循环

用 🤗 Evaluate 累积所有 batch 的预测，最后一次性算指标：

import evaluate
metric = evaluate.load("glue", "mrpc")

model.eval()
for batch in eval_dataloader:
    batch = {k: v.to(device) for k, v in batch.items()}
    with torch.no_grad():                  # 评估不计算梯度
        outputs = model(**batch)
    logits = outputs.logits
    predictions = torch.argmax(logits, dim=-1)
    metric.add_batch(predictions=predictions, references=batch["labels"])

metric.compute()
# {'accuracy': 0.843, 'f1': 0.891}
07

Accelerate 与总结 分布式训练 / 本章回顾

上面的循环只在单 CPU/GPU 跑。借助 🤗 Accelerate，几处改动就能扩展到多 GPU / TPU：

三处关键改动

+ from accelerate import Accelerator
  ...
+ accelerator = Accelerator()   # 探测环境，初始化分布式设置

  model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
  optimizer = AdamW(model.parameters(), lr=3e-5)

- device = torch.device("cuda") if torch.cuda.is_available() else torch.device("cpu")
- model.to(device)
+ train_dl, eval_dl, model, optimizer = accelerator.prepare(
+     train_dataloader, eval_dataloader, model, optimizer)   # 用适当容器包装

  ...
  for batch in train_dl:
-     batch = {k: v.to(device) for k, v in batch.items()}   # Accelerate 已托管设备迁移
      outputs = model(**batch)
      loss = outputs.loss
-     loss.backward()
+     accelerator.backward(loss)   # 替换为 accelerate 版反向传播

① 实例化 Accelerator

自动探测环境并初始化分布式设置。

② accelerator.prepare()

把 DataLoader / model / optimizer 包装进合适的容器。

③ accelerator.backward()

替换 loss.backward()，接管跨设备梯度同步。

🚀
运行方式 把脚本存为 train.py，用 accelerate launch train.py 启动分布式训练（首次会问几个配置问题）。Notebook 里可用 notebook_launcher(training_function)。
