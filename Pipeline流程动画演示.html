NLP Pipeline 完整流程

深入理解 Transformer 模型的三大处理步骤

一个完整的 NLP 任务处理流程包含三个关键阶段：将原始文本转换为模型输入、通过神经网络计算、最后转换为可读结果。

① 预处理
② 模型计算
③ 后处理

第一步：预处理 (Tokenization)

将原始文本转换为模型能理解的数字形式。

处理流程

输入： 文本字符串

text = "I love this movie"

第1阶段 - 分词 (Tokenization)：

将句子拆分成 tokens（词或子词）

tokens = ["[CLS]", "I", "love", "this", "movie", "[SEP]"] # 长度：6 tokens

第2阶段 - ID 映射：

将每个 token 转换为词汇表中的整数 ID

input_ids = [101, 1045, 2572, 2054, 3007, 102] # 词汇量：30,522

第3阶段 - 张量转换：

转换为 PyTorch 张量以供模型使用

tensor_ids = torch.tensor([input_ids]) # Shape: [1, 6] (1个句子，6个tokens)
关键概念：
• [CLS] = Classification token，句子的特殊开始标记
• [SEP] = Separator token，句子结束标记
• Padding = 短句子填充到固定长度
• Attention mask = 标记有效token和填充token

第二步：模型计算 (Forward Pass)

通过 Transformer 编码器逐层处理向量，提取语义特征。

① Token Embedding - 向量化

将整数 ID 转换为 768 维的密集向量

# 输入 (Input IDs) input_ids = [101, 1045, 2572, 2054, 3007, 102] # Shape: [1, 6] # Embedding 层映射 embedded = embedding(input_ids) # Shape: [1, 6, 768] # 每个token变成768维向量

关键参数： 嵌入维度=768，词汇量=30,522

输出： 6 个 768 维向量的序列

② Self-Attention - 多头注意力

12 个注意力头并行学习 tokens 之间的依赖关系

# 输入 x = [1, 6, 768] # 6个token，每个768维 # 12个并行注意力头 # 每头处理 768/12 = 64 维 attention_output = multi_head_attention(x) # Shape: [1, 6, 768] 维度保持不变 # 每个token都关注其他所有tokens # 学习token间的语义关联

注意力头的作用：

Token 0: [CLS]
Token 1: I
Token 2: love
Token 3: movie

说明： 每个色块代表一个注意力头，颜色越深表示该头的注意力权重越大；鼠标悬停可查看具体权重数值。

③ Pooling - 序列汇聚

从 6 个 tokens 的矩阵中提取单个全局表示向量

# 输入 (Self-Attention后) x = [1, 6, 768] # 6个tokens，每个768维 # Pooling 操作：取 [CLS] token pooled = x[:, 0, :] # 只取第1个token的向量 # Shape: [1, 768] # [CLS] token聚合了全句的语义信息

为什么选 [CLS]？

Transformer 在预训练时已学会将 [CLS] token 用作整个句子的表示，它聚合了所有其他 tokens 通过自注意力传递来的信息。

④ Classification - 分类层

线性变换将 768 维向量映射到分类标签数

# 输入 (Pooled向量) pooled_output = [1, 768] # 分类层 (线性层) classifier = Linear(768, 2) # 2分类：NEGATIVE/POSITIVE logits = classifier(pooled_output) # Shape: [1, 2] # 输出示例 logits = [-2.5, 3.2] # 正面分数远高于负面分数

参数量： 768 × 2 + 2 = 1,538 个参数

输出： 原始预测分数 (logits)，未经过概率转换

完整路径： [101, 1045, ...] → 768维向量矩阵 → 注意力加权 → [CLS]表示向量 → 线性映射 → Logits [-2.5, 3.2]

第三步：后处理 (Post-Processing)

将模型的原始输出转换为人类可读的预测结果。

① Logits - 原始分数

模型的直接输出，未经归一化

logits = [-2.5, 3.2] # logits[0] = NEGATIVE 的分数 = -2.5 # logits[1] = POSITIVE 的分数 = 3.2

② SoftMax - 概率转换

将 logits 转换为 0-1 范围的概率

import torch.nn.functional as F logits = tensor([-2.5, 3.2]) probabilities = F.softmax(logits, dim=-1) # 计算过程 e^(-2.5) = 0.082 e^(3.2) = 24.533 sum = 0.082 + 24.533 = 24.615 # 归一化 p_negative = 0.082 / 24.615 = 0.003 (0.3%) p_positive = 24.533 / 24.615 = 0.997 (99.7%)

③ 取最大 & 映射标签

找最高概率的类别并映射到标签名称

probabilities = [0.003, 0.997] # 取最大值索引 pred_idx = argmax(probabilities) # 结果：1 # 映射到标签 id2label = {0: "NEGATIVE", 1: "POSITIVE"} label = id2label[pred_idx] # "POSITIVE" confidence = probabilities[pred_idx] # 0.997

最终输出：

result = { 'label': 'POSITIVE', 'score': 0.9973 }
解释： 模型以 99.73% 的置信度预测该评论为正面情感

完整代码示例

方式1：使用 Hugging Face Pipeline API (推荐)

最简洁的方式，自动完成三个步骤：

from transformers import pipeline # 初始化 pipeline classifier = pipeline("sentiment-analysis") # 执行完整流程 # (预处理 → 模型计算 → 后处理) result = classifier("I love this movie") print(result) # 输出: # [{'label': 'POSITIVE', 'score': 0.9973}]

方式2：手动实现完整流程

逐步控制每个阶段：

from transformers import ( AutoTokenizer, AutoModelForSequenceClassification ) import torch import torch.nn.functional as F # ========== 第一步：预处理 ========== tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased") text = "I love this movie" inputs = tokenizer(text, return_tensors="pt") # ========== 第二步：模型计算 ========== model = AutoModelForSequenceClassification.from_pretrained( "bert-base-uncased", num_labels=2 ) outputs = model(**inputs) logits = outputs.logits # Shape: [1, 2], e.g., [[-2.5, 3.2]] # ========== 第三步：后处理 ========== probabilities = F.softmax(logits, dim=-1) pred_idx = torch.argmax( probabilities, dim=-1 )[0].item() confidence = probabilities[0][pred_idx].item() label = model.config.id2label[pred_idx] print(f"Label: {label}") print(f"Confidence: {confidence:.4f}") # 输出: Label: POSITIVE, Confidence: 0.9973
