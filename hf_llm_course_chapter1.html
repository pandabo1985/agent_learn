什么是自然语言处理 Chapter 1/2

NLP 是语言学与机器学习的交叉领域，核心目标不仅是识别单词，更是理解其上下文含义。

句子级任务

  • 情感分析
  • 垃圾邮件检测
  • 语法正确性判断
  • 文本相似度计算

词级任务

  • 词性标注（POS）
  • 命名实体识别（NER）
  • 遮蔽词填充（MLM）

生成任务

  • 文本补全与续写
  • 文本摘要生成
  • 代码生成

跨序列任务

  • 机器翻译
  • 问答系统
  • 信息提取
⚠️
为什么困难？ 语言具有固有的歧义性与多层语义——人类能轻松判断句子相似度，但为机器设计直接算法处理此类问题存在显著障碍。NLP 还延伸至语音识别与计算机视觉等领域。

Transformers 能做什么 Chapter 1/3

pipeline() 将模型与预处理、后处理步骤自动串联，使用者直接输入文本即可获得结果。

from transformers import pipeline

# 情感分析——最简用法
classifier = pipeline("sentiment-analysis")
result = classifier("I love Hugging Face!")
# [{'label': 'POSITIVE', 'score': 0.9998}]

# 零样本分类——无需微调即可使用自定义标签
clf = pipeline("zero-shot-classification")
clf("This is a Python tutorial",
    candidate_labels=["education", "tech", "sports"])

# 文本生成——控制输出数量与长度
gen = pipeline("text-generation", model="gpt2")
gen("In this course, we will teach you how to",
    max_length=30, num_return_sequences=2)

Transformer 工作原理 Chapter 1/4

Transformer 架构自 2017 年问世，以注意力机制为核心，在 NLP 领域引发范式革命。

关键发展里程碑

2017
原始 Transformer 论文发布
Vaswani 等人提出「Attention Is All You Need」，奠定现代 NLP 基础。
2018
GPT 与 BERT 相继发布
OpenAI 发布 GPT（自回归），Google 发布 BERT（自编码），奠定两大流派。
GPTBERT
2019
变体与压缩模型涌现
模型规模与效率并行探索，轻量级方案逐步成熟。
GPT-2DistilBERTBARTRoBERTa
2020+
超大规模模型时代
GPT-3 等模型展示涌现能力，Seq2Seq 系列全面成熟。
GPT-3T5mBART

核心创新：注意力机制

处理每个词时权衡句子中所有其他词的重要程度，从而捕捉长距离语义依赖关系。

训练范式：迁移学习

先在海量语料上做无监督预训练，再通过少量标注数据进行微调适配下游任务。

📖
三个易混淆术语 架构 — 模型结构框架（层数、维度等）； 检查点 — 某次训练后保存的具体权重； 模型 — 两者的统称，视语境而定。

Transformer 架构全景

Transformer
├── 编码器（Encoder）
├── 双向自注意力层
└── 前馈神经网络
├── 解码器（Decoder）
├── 单向（因果）自注意力层
├── 交叉注意力层（Cross-Attention）
└── 前馈神经网络
└── 三种组合方式
├── 仅编码器 → BERT 家族
├── 仅解码器 → GPT 家族
└── 编码器-解码器 → T5 / BART 家族
05

编码器模型 Chapter 1/5

仅使用 Transformer 编码器部分，具备双向注意力——每个词都能同时关注前后所有单词。

核心特征

  • 双向注意力（全局上下文可见）
  • 自编码（Auto-Encoding）训练范式
  • 预训练：随机遮盖词语后恢复（MLM）

最适合的任务

  • 句子分类（情感分析）
  • 命名实体识别（NER）
  • 阅读理解 / 问答（抽取式）
  • 词性标注（POS）

代表模型

BERT RoBERTa DistilBERT ALBERT ELECTRA
💡
为什么双向很重要？ 理解「我喜欢苹果」中「苹果」的含义，需要同时参考前文「我喜欢」和后文语境，单向模型无法做到这点。
06

解码器模型 Chapter 1/6

仅使用 Transformer 解码器部分，采用单向（因果）注意力——每个词只能看到它之前的词。

核心特征

  • 单向注意力（屏蔽未来信息）
  • 自回归（Auto-Regressive）训练范式
  • 预训练：预测下一个词（CLM）

最适合的任务

  • 文本生成（续写、对话）
  • 代码补全
  • 故事创作

代表模型

GPT GPT-2 GPT-3 CTRL Transformer XL
💡
自回归生成原理 每次预测一个词，将该词拼入序列后再预测下一个词，如此循环直至生成终止符或达到最大长度。
07

编码器-解码器模型 Chapter 1/7

同时使用编码器与解码器，又称序列到序列（Seq2Seq）模型，最适合根据输入生成全新输出序列的任务。

工作机制

  • 编码器：双向注意力，理解整个输入
  • 解码器：单向注意力，逐步生成输出
  • 交叉注意力（Cross-Attention）连接二者

最适合的任务

  • 摘要生成
  • 机器翻译
  • 生成式问答
  • 文档改写
🔬
T5 的预训练策略 用「单个掩码特殊词」替换随机文本范围（而非逐词遮盖），再恢复被替换的内容——比 BERT 式 MLM 更高效地利用训练信号。

代表模型

T5 BART mBART Marian

偏见与局限性 Chapter 1/8

预训练模型从互联网海量文本中学习，不可避免地吸收其中的社会偏见与刻板印象，且微调无法完全消除这种固有偏见。

以 BERT（训练数据：维基百科 + BookCorpus）的完形填空为例：

This man works as a [MASK].
  • 律师（lawyer）
  • 木匠（carpenter）
  • 医生（doctor）
  • 服务员（waiter）
  • 机械师（mechanic）
This woman works as a [MASK].
  • 护士（nurse）
  • 女服务员（waitress）
  • 教师（teacher）
  • 女仆（maid）
  • 妓女（prostitute）
🚨
关键结论 即使使用了相对中立的数据集，训练数据蕴含的性别职业刻板印象依然被模型习得并放大。在生产环境部署模型时，必须建立偏见检测与缓解机制，不能假定预训练模型是「中立」的。

章节总结 Chapter 1/9

三大模型家族速查对比——根据任务类型选择合适架构是关键决策。

类型 注意力方向 训练范式 代表模型 典型任务
编码器
Encoder-Only		 双向 自编码（MLM） BERT RoBERTa DistilBERT 分类、NER、问答
解码器
Decoder-Only		 单向（因果） 自回归（CLM） GPT-2 GPT-3 CTRL 文本生成、对话
编码器-解码器
Seq2Seq		 混合（交叉注意力） 遮蔽文本恢复 T5 BART Marian 翻译、摘要、生成式 QA

⚠️ 注意事项

  • 模型存在社会偏见，需检测与缓解
  • 微调不能消除预训练引入的偏见
  • 区分架构、检查点与模型三个概念
  • 任务类型决定最优架构选择
