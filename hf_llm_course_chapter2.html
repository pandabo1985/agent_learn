01

本章简介 为何需要 🤗 Transformers

Transformer 模型动辄数百万到数十亿参数，训练与部署复杂；新模型层出不穷且各有实现。🤗 Transformers 用一个统一 API 解决「加载、训练、保存任意模型」的问题。

三大设计特点

🚀 易于使用

仅需两行代码即可下载、加载并使用先进的 NLP 模型进行推理。

🧩 灵活

所有模型本质都是 nn.Module / tf.keras.Model，可像普通模型一样处理。

📖 简单

「全在一个文件中」：每个模型前向传播独立定义，便于理解与修改，互不影响。

🧭
本章学习路径 端到端复刻 pipeline() → 模型 API（Config / Model）→ tokenizer API → batch / padding / 编码解码。
⚠️
提示 为充分利用 Model Hub 与 🤗 Transformers 的所有功能，建议先创建一个 HuggingFace 账户。

→

→
🧠

→
🔢

① 使用 Tokenizer 预处理

Transformer 无法直接处理原始文本，tokenizer 负责：拆分为 token、映射为 input ID、添加特殊标记（如 [CLS] / [SEP]）。预处理方式必须与模型预训练时完全一致。

from transformers import AutoTokenizer

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

raw_inputs = ["I've been waiting for a HuggingFace course my whole life.",
              "I hate this so much!"]
# padding 对齐长度，truncation 防止过长，return_tensors 指定张量框架
inputs = tokenizer(raw_inputs, padding=True, truncation=True, return_tensors="pt")
💡
关键产物 输出字典含 input_ids（每行一句话的 token ID）与 attention_mask（标记哪些位置是真实 token、哪些是填充）。

② 探索模型与模型头

AutoModel 只输出隐状态（hidden states，高维特征向量）。模型输出通常有三个维度：

Batch size

一次处理的序列数（示例中为 2）

Sequence length

序列长度（示例中为 16）

Hidden size

每个输入的向量维度（小模型 768，大模型可达 3072+）

要得到具体任务的结果，需加载带「模型头」的类——例如序列分类用 AutoModelForSequenceClassification，它把高维向量压缩成每个标签一个分数。

from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained(checkpoint)
outputs = model(**inputs)
print(outputs.logits.shape)  # 2 句 × 2 标签 → torch.Size([2, 2])

③ 后处理：logits → 概率

模型输出的是未归一化的 logits，需经 SoftMax 转成概率（Transformers 训练时通常把最后激活与损失融合，故输出仍为 logits）：

import torch
predictions = torch.nn.functional.softmax(outputs.logits, dim=-1)
# 通过 config.id2label 把 ID 映射回标签：{0:'NEGATIVE', 1:'POSITIVE'}

至此，我们用 tokenizer 预处理 + 模型 + 后处理，手工复刻了 pipeline() 的全部逻辑。

03

模型 API Config / 加载 / 保存 / 推理

AutoModel 是对各类模型的智能包装，能根据 checkpoint 自动推断架构并实例化；若已知具体类型，也可直接用对应类（如 BertModel）。

创建模型与 Config

from transformers import BertConfig, BertModel

config = BertConfig()   # 默认配置 → 随机初始化（输出胡言乱语，需训练）
model  = BertModel(config)
# 关键属性：hidden_size=768, num_hidden_layers=12, num_attention_heads=12 ...

三种加载与保存方式

方式说明产物
随机初始化 BertModel(BertConfig())，可运行但输出无意义 未训练权重
from_pretrained() 加载预训练 checkpoint，权重缓存到 ~/.cache/huggingface（可用 HF_HOME 自定义） 可直接推理 / 微调
save_pretrained() 把模型写入磁盘目录 config.json + model.safetensors
🔑
两个文件相辅相成 config.json 描述架构（含元数据与 Transformers 版本）；model.safetensors 即「状态字典」，存放全部权重。缺一不可。

用张量做推理

模型只接受数字（由 tokenizer 转换的张量）。多个编码序列需先组成矩形张量：

import torch
sequences = ["Hello!", "Cool.", "Nice!"]
encoded   = [[101, 7592, 999, 102], ...]  # tokenizer 给出的 input IDs
model_inputs = torch.tensor(encoded)
output = model(model_inputs)   # 必填项只有 input IDs
04

Tokenizers 三种分词算法 + 编解码

tokenizer 的目标：把文本转换为模型可处理的数字，且尽量有意义、尽量简洁。

三种分词算法对比

算法做法优点缺点
基于单词 Word-based 按空格/规则切词，如 split() 简单直观 词汇表巨大；dog/dogs 视为无关；多 [UNK]
基于字符 Character-based 拆分为单个字符 词汇表小；几乎无未知 token 单字符语义弱；序列变长（一词≈10+ token）
基于子词 Subword 常用词保留，罕见词拆为有义子词 小词汇表 + 高覆盖；语义保留 需配合算法（BPE / WordPiece 等）
🔤
子词示例 tokenization → token + ##ization；annoyingly → annoying + ly。两个子词都更常见，且复合后语义完整。

Byte-level BPE

用于 GPT-2

WordPiece

用于 BERT

SentencePiece / Unigram

用于多语言模型

加载 / 保存

from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained("bert-base-cased")
tokenizer.save_pretrained("directory_on_my_computer")
# from_pretrained / save_pretrained 同时保存算法(≈架构)与词汇表(≈权重)

编码（Encoding）= 分词 + 转 ID

sequence = "Using a Transformer network is simple"
tokens = tokenizer.tokenize(sequence)
# ['Using', 'a', 'transform', '##er', 'network', 'is', 'simple']
ids = tokenizer.convert_tokens_to_ids(tokens)
# [7993, 170, 11303, 1200, 2443, 1110, 3014]

解码（Decoding）= ID → 字符串

tokenizer.decode([7993, 170, 11303, 1200, 2443, 1110, 3014])
# 'Using a Transformer network is simple' —— 会自动合并同词的子词
05

处理多个序列 Batch / Padding / Mask

🐞
常见坑：单句直接送入模型报错 model(input_ids) 会抛 IndexError: Dimension out of range——因为模型默认需要一个句子列表，必须补上 batch 维度 torch.tensor([ids])。

① 模型需要一批输入

批处理（Batching）是一次性送入多个句子。即便只有一句，也要构造成 batch 维度。

② 填充 Padding

张量必须是矩形，长度不同的句子无法直接堆叠。用 tokenizer.pad_token_id 把短句补齐到相同长度。

⚠️
填充会污染预测！ 注意力层会把填充 token 也纳入上下文，导致 batch 中短句的 logits 与单独预测时不同。必须用 attention mask 告诉模型忽略填充。

③ 注意力掩码 Attention Mask

与 input IDs 形状相同的张量：1 表示关注，0 表示忽略（即填充位）。

batched_ids = [[200, 200, 200],
               [200, 200, tokenizer.pad_token_id]]
attention_mask = [[1,1,1],
                  [1,1,0]]   # 第二句末位是填充 → 0
outputs = model(torch.tensor(batched_ids), attention_mask=torch.tensor(attention_mask))
# 加上 mask 后，batch 中短句的 logits 与单独预测一致

④ 更长的句子

🚧 模型最大序列长度

多数模型只支持 512 或 1024 个 token，超出会崩溃。

✂️ 两种对策

①换用长序列模型（Longformer、LED）；②截断 sequence[:max_sequence_length]。

06

综合应用 一把梭的高级 API

前面手动完成了分词、转 ID、填充、截断、mask。其实直接调用 tokenizer 对象就能一次搞定所有：

from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

model_inputs = tokenizer(sequence)            # 单句
model_inputs = tokenizer(sequences)           # 多句，API 一致
model_inputs = tokenizer(sequences, padding="longest")      # 填到最长
model_inputs = tokenizer(sequences, padding="max_length", max_length=8)
model_inputs = tokenizer(sequences, truncation=True)         # 截断
model_inputs = tokenizer(sequences, padding=True, return_tensors="pt")  # "pt"/"tf"/"np"

特殊 Tokens 自动添加

直接调用 tokenizer 时，它会在首尾自动加上预训练时用到的特殊标记，结果与手动 tokenize() 略有不同：

[101] [CLS] ← 开头自动添加
[1045, 1005, 2310, ...] ← 实际 token IDs
[102] [SEP] ← 结尾自动添加
解码 → "[CLS] i've been waiting ... [SEP]"
不同模型加法不同；tokenizer 知道哪些必需并会自动处理

小结：从 tokenizer 到模型

import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer  = AutoTokenizer.from_pretrained(checkpoint)
model      = AutoModelForSequenceClassification.from_pretrained(checkpoint)
sequences  = ["I've been waiting for a HuggingFace course my whole life.", "So have I!"]

tokens = tokenizer(sequences, padding=True, truncation=True, return_tensors="pt")
output = model(**tokens)   # 多序列(填充) + 长序列(截断) + 指定张量，一步到位
