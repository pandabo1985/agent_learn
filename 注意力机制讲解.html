注意力机制 深度讲解

从动机到原理 · 从 Q/K/V 到多头注意力 · 读懂 Transformer 的核心

01

什么是注意力机制

Attention 权重分配 上下文向量 聚焦

直觉理解

人类阅读一个句子时，不会平等对待每个词——读到"银行"时，会根据上下文判断是"金融机构"还是"河岸"。注意力机制让神经网络也能做到这一点：动态地为不同位置的信息分配不同的重要性权重，而不是固定地处理每个位置。

技术定义

注意力机制是一种计算加权求和的方法：给定一组候选信息（Value），根据查询（Query）与每条候选信息的键（Key）的相似度，计算出权重，再对 Value 加权求和，得到一个聚焦后的上下文向量。

一句话：注意力机制 = "根据当前需求，动态决定应该重点关注哪些信息，然后加权汇总"。
02

为什么需要注意力机制

瓶颈向量 长距离依赖 梯度消失 信息压缩

RNN 的瓶颈问题

传统的 Seq2Seq 模型（编码器-解码器结构）将整个输入序列压缩成一个固定长度的向量，再由解码器展开。这个"瓶颈向量"必须承载所有信息，句子越长，早期信息丢失越多，翻译质量下降越明显。

Seq2Seq 的瓶颈问题
├─ 编码器：逐词处理 → 压缩成一个固定向量 c
│  └─ 问题：长句子中，第 1 个词的信息到最后几乎消失
│
├─ 解码器：只能看到这一个向量 c
│  └─ 问题：无论生成哪个目标词，都用同一个 c，缺乏针对性
│
└─ 注意力机制的解法：让解码器在生成每个词时，
   都能"回头看"编码器的所有隐状态，动态聚焦相关部分
动机总结：注意力机制打破了"固定瓶颈向量"的限制，让模型在处理每个输出时，都能有针对性地访问输入序列的任意位置，从根本上解决了长距离依赖问题。
03

核心计算：Query / Key / Value

Query 查询 Key 键 Value 值 Softmax 缩放点积

三个角色的类比

可以把注意力机制类比成图书馆检索：你提出一个查询（Query），图书馆里每本书都有一个索引标签（Key），你根据查询与标签的匹配程度决定借哪些书，最终取出书的内容（Value）加权合并成答案。

缩放点积注意力（Scaled Dot-Product Attention） Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

逐步拆解

① Q·Kᵀ：计算查询与所有键的点积相似度，得到原始注意力分数矩阵。
② ÷√dₖ：除以键向量维度的平方根，防止点积值过大导致 softmax 梯度消失。
③ softmax：将分数归一化为概率分布（权重之和为 1）。
④ ·V：用权重对 Value 加权求和，得到聚焦后的输出向量。

为什么要除以 √dₖ？当维度 dₖ 很大时，点积的方差会随维度增大，导致某些分数极大、softmax 后几乎全部权重集中在一个位置，梯度几乎为零。除以 √dₖ 可将方差稳定在合理范围。
04

自注意力（Self-Attention）

Self-Attention 同源 Q/K/V 全局关联 并行计算

什么是自注意力

普通注意力的 Query 来自解码器，Key/Value 来自编码器（跨序列）。自注意力则让同一序列中每个位置都对自身序列内的所有位置做注意力——Q、K、V 均来自同一输入，通过三个不同的线性投影矩阵（Wq、Wk、Wv）生成。

输入序列：["我", "爱", "学习"]
├─ 每个词 → 线性变换 → Q、K、V 向量
│
├─ "我" 的 Q 与 ["我","爱","学习"] 的所有 K 计算相似度
│  → 得到权重：[0.7, 0.2, 0.1]
│  → 加权求和所有 V → "我"的输出向量（融合了全局上下文）
│
├─ "爱" 同理，关注不同的位置组合
│
└─ 三个位置可同时并行计算，无需按序等待

自注意力的核心优势

全局感受野：任意两个位置之间的距离都是 1 步（直接计算），不像 RNN 需要逐步传递。
训练可并行：序列中所有位置的注意力可同时计算，GPU 利用率极高。
动态上下文：同一个词在不同语境中会产生不同的注意力权重，捕捉多义性。

对比 RNN：RNN 中"我"对"学习"的影响需经过两步状态传递，路径越长信息越易丢失；自注意力中"我"和"学习"直接交互，路径长度恒为 1。
05

多头注意力（Multi-Head Attention）

多头 子空间 并行注意力 拼接投影

为什么需要多头

单头注意力在每次计算中只能关注一种模式（比如语法关系）。多头注意力将 Q/K/V 分别投影到 h 个不同的低维子空间，每个"头"独立学习不同类型的关联（语义、语法、指代、位置等），最后拼接所有头的输出再做一次线性投影。

Multi-Head Attention
├─ head₁ = Attention(QW₁q, KW₁k, VW₁v)  ← 关注语义相似性
├─ head₂ = Attention(QW₂q, KW₂k, VW₂v)  ← 关注句法依存
├─ head₃ = Attention(QW₃q, KW₃k, VW₃v)  ← 关注指代关系
│  … (共 h 个头，Transformer 原文 h=8)
│
└─ 输出 = Concat(head₁,…,headₕ) · Wₒ
多头注意力公式 MultiHead(Q,K,V) = Concat(head₁, …, headₕ) · Wₒ
类比：就像用 8 个不同角度的摄像头同时拍摄同一场景，每个摄像头捕捉不同侧面的信息，最后合并成一张全面的理解图。
06

发展历程

2014 · Bahdanau Attention

Bahdanau 等人提出第一个神经机器翻译注意力机制，让解码器在生成每个词时对编码器所有隐状态加权求和，显著提升了长句翻译质量。这是"注意力机制"的起点。

2015 · Luong Attention

Luong 等人提出更简洁的点积注意力和 local attention 变体，进一步验证了注意力机制的有效性，并将计算效率提升。

2017 · Transformer（"Attention Is All You Need"）

Vaswani 等人提出完全基于自注意力的 Transformer 架构，彻底抛弃 RNN，引入缩放点积注意力、多头注意力和位置编码。成为现代 NLP 的奠基之作。

2018–2019 · BERT / GPT 系列

BERT（双向 Transformer 编码器）和 GPT（单向 Transformer 解码器）证明预训练大规模 Transformer 后微调可以在几乎所有 NLP 任务上取得最优结果，注意力机制成为 AI 主流范式。

2022 · Flash Attention

Dao 等人提出 Flash Attention，通过 IO 感知的分块计算重写了注意力的 GPU 实现，将内存占用从 O(n²) 降至 O(n)，训练速度提升 2–4×，使超长上下文窗口成为可能。

2023–今 · 长上下文与稀疏注意力

Flash Attention 2/3、Ring Attention、Sliding Window Attention 等变体持续突破上下文长度限制。主流模型上下文窗口从 2K token 扩展至 128K–1M token。

07

注意力机制主要变体

全局注意力 稀疏注意力 线性注意力 Flash Attention 分组查询注意力
变体核心思路解决问题代表应用
全局自注意力 每个位置与所有位置计算注意力 最强建模能力，无近似损失 BERT、GPT-2、原始 Transformer
稀疏注意力 只与局部窗口或固定步长位置计算 将复杂度从 O(n²) 降低 Longformer、BigBird
线性注意力 用核函数近似 softmax，改变计算顺序 理论 O(n) 复杂度 Performer、cosFormer
Flash Attention IO 感知分块计算，不近似 softmax 显存从 O(n²) 降至 O(n)，速度大幅提升 GPT-4、Llama 3、Claude 等
分组查询注意力（GQA） 多个 Query 头共享一组 K/V 头 推理时 KV Cache 显著减小 Llama 2/3、Mistral
多查询注意力（MQA） 所有 Query 头共享同一对 K/V 推理速度最快，KV Cache 最小 PaLM、Falcon
趋势：标准全局注意力仍是训练质量的上限，Flash Attention 已成为工程标配，GQA 正在快速取代 MHA 成为推理优化的主流选择。
